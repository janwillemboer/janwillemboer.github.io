2. Consume External Data

Advanced applications often use data from external data sources, like data sources from the government. In this track you will use external data and integrate it into your own application. This track is the most "functional" track, because the data has to have a real, logical use in your application.

  • The external system must be complex enough, at the discretion of the teacher. A postal code check or a list of millimeters of rain for the coming hours probably won't suffice, but something that shows available parking spaces on a map probably will.
  • The system must integrate nicely with the external data / API. If you build a system that is a chat client for elderly people and also integrates an agricultural map, that doesn't make much sense.
  • Your application can handle latency or the service being offline, with fast and clear communication to the user.
  • You test this using an automated test suite.
  • There is a mock of the service for testing the functionality of your application in automated tests.
  • There needs to be a way to warn an administrator when the service goes down.
  • There needs to be a user interface to configure the service, for example to set timeouts and to set the URL in case of an external service, and to configure the warning system.
  • It is also possible to choose data that is not available as an online API. In that case you download the data to your own system and write an API yourself to expose the data. This requires you to design an additional, consistent API for that dataset next to your own application.
  • Documentation. All data, all conversions, and all features in your advanced track should be described in detail in the technical design. A manual should provide information to admins on how to use the admin part.

A few hints for external sources:

  • https://data.overheid.nl/ has all kinds of dutch government databases,
  • https://opendata.rdw.nl/ has data from the RDW, the dutch vehicle registry,
  • https://www.kadaster.nl/zakelijk/datasets/open-datasets contains data from the Land Registry (Kadaster),
  • https://www.pdok.nl/services-en-api-s has dutch mapping APIs, for example the dutch aerial photos.
  • https://en.wikipedia.org/api/rest_v1/ is an API that exposes wikipedia data,
  • https://data.europa.eu datasets from the EU, etc etc.

Common ways to expose APIs are SPARQL, REST and GraphQL. If you search for that term, you will at least know for sure that it is an API and not a download.

Basic requirements for grading

  • An external datasource is used and integrated in the application
  • There is a show-and-tell document to prove the system works.
